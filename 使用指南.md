# 星猫塔罗牌 (Star Cat Tarot) - 项目使用与维护指南

这份文档记录了项目中所有必须要用到的操作指令和开发流程，方便后续查阅与快捷操作。未来如果加入了新的脚本指令，也应同步记录于此。

---

## 🃏 一、如何导入 / 新增一副塔罗牌？

当你收集到了新塔罗牌的高清图片集合时（例如提取到了一个文件夹里），只需要执行**一步操作**即可全自动处理并导入系统。

### 前置要求：
1. 你必须将这副牌的所有图片放到电脑里的**同一个文件夹**里。
2. 文件夹内必须有一张名为 `back` (比如 `back.png` 或 `back.jpg`) 的图片，作为**牌背**。
3. 其余只要是 `.png` / `.jpg` / `.jpeg` 后缀的，都会被系统识别为**牌面**。

### 核心执行指令：
打开终端，务必确当前处于项目根目录（`tarot-app`），然后执行：
```bash
npm run add-deck <牌组名称> <源素材文件夹的绝对路径或相对路径>
```

**示例（比如要把桌面名叫 `new_cards` 的相册导入进去，命名为 `dark-tarot`）**：
```bash
npm run add-deck dark-tarot ~/Desktop/new_cards
```

### 脚本全自动处理细节说明：
当执行此指令后，系统底层的 `/scripts/add-deck.sh` 脚本将会替你处理如下繁杂事物：
* 自动创建 `public/decks/<牌组名>` 文件夹。
* 将几 MB、十几 MB 的庞大高清 `PNG` 素材无损转化为利于网络传输的 `JPEG` 进行压缩（极大降低网页内存崩溃概率）。
* 自动清洗包含中文、加号（+）、空格等特殊字符的非法文件名。
* 自动提取数据库、触发代码中的渲染层，使其瞬间显示在界面菜单中。

## 🌐 二、如何将最新内容同步到线上（发布网站）？

在你完成了对网页的任何修改（如：加了新牌组、修改了CSS配色、修了Bug）并满意本地预览效果之后，必须要同步出去，线上的移动端才会更新。

这里提供**两种**常用的提交流程方案：

### 方案 A：通过 GitHub 自动同步发布（推荐长期保留法）

这个方法会将修改后的源代码与图片**同时作为备份**存档至 GitHub 的云端，并且由于 Vercel 配置了监控，推流的同时线上网页会自动构建，几十秒极速上线。

1. **第一步：把所有的系统改动收集并装箱**
   ```bash
   git add .
   ```
2. **第二步：给这次改动贴个备注标签**
   ```bash
   git commit -m "feat: 这里写写你改了什么，例如：新增了xxxx塔罗牌配置"
   ```
3. **第三步：将包裹正式发送到 GitHub 云服务器**
   ```bash
   git push
   ```
> *注：完成后无需任何手动操作，Vercel 后台会自动嗅探变动并部署。刷新线上网页即可。*

---

### 方案 B：本地直传 Vercel（快速暴力覆盖法）

如果 GitHub 那天抽风宕机、端口又被墙封死，或者单纯嫌上面的步骤麻烦。
你可以跳过代码云盘备份，在终端直接跑这一行指令将网页打包发往 Vercel。

```bash
npx vercel --prod
```
静待打包完毕并出现绿色的 `✅ Production` 网址即可。

## 💻 三、如何进行日常的开发和调试？

当你关机后重新回来开发，或者想在推送到线上之前看看长什么样，需要启动**本地开发服务器**。

**启动指令：**
```bash
npm run dev -- --host
```

* 等待终端输出带有绿色 `Local: http://localhost:5173` 后，按住键盘里的 Command 键并点击鼠标，即可在浏览器直接预览你的项目。
* 命令里的 `--host` 支持让你用处于同一个 WiFi 局域网下的手机，通过输入类似 `http://192.168.x.x:5173` 的地址实时看到手机页面的真实浏览效果。
